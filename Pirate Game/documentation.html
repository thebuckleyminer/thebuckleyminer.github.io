The Legend of Pirate Island

Documentation of how my games satisfies the requirements

20 Game grid that uses CSS and is generated by JavaScript

The game grid is generated in JavaScript. It chooses unique words and color for every card. With CSS the word placement on the card takes place.

20 User input in the form of text, select options, and/or buttons

The four players have a text entry box to input their names, as well as a button to select their role, there is also finish turn buttons, a clue text input, a number list dropdown to choose quantity and more!

20 User input in the form of mouse clicks

When a card is clicked it changes the card from the front side to the back side.

20 Dynamic modification of HTML using innerHTML or DOM functions

The whole game grid is generated, as well as the pre-game tutorial menus, and name selection menus. The main part of the dynamic modification though is the score, found player, and players turn parts of the game page that change as the game progresses.

20 Use of XMLHttpRequest to load JSON or XML game data that is displayed on the grid Page

The JSON File I have chosen to include for this project contains the word list which is an array that holds the card words. I also have in the JSON the first name list and last name list.

20 Incorporates HTML 5 audio tags, video tags, canvas, or local storage

For this requirement I added an audio file on the Game Grid page. This audio file is the only source not personally created for this semester project besides the three fonts I have used. The audio track is called: "♩♫ Epic Pirate Music ♪♬ - Pirate Crew (Copyright and Royalty Free)" and can be found at https://youtu.be/hVBgKCYrI-c The audio will auto load and loop.

120 Game logic code

In this four player game, the main game logic parts include:
  • Dynamic name input for all four players.
  • Generated tiles.
  • The starting team has 9 cards and the other team has 8.
  • The scores update when a card is chosen.
  • There are two player modes, one that gives a clue, and one that guesses.
  • There are two teams, so four player roles total.
  • There is a next player button on a screen that hides the cards so other players are not revealed the cards.
  • The Captain's clue is displayed on the page.
  • When a player clicks on a card it reveals the back side.

20 Extra Credit Include a second HTML5 component. If your first HTML5 component is media (audio or video) then your second component can't be a media component.

For this requirement I implemented the HTML 5 local storage for the purposes if the players want to play another game. This works by storing their names and a restart value to local storage. When the game is now rebooted the game bypasses the tutorial and player name sections stright to a freshly generated board. The local storage is then cleared immediatly after loaded, so when the game is refreshed or is opened at a later date it will start fresh and not load in this manner. Thus the restart game only works when the restart game button is pressed. 250 TOTAL
